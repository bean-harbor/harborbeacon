# 2026-04-10 Codex

## 今日目标

- 将项目目标从通用本地 Agent 收敛为面向 HarborOS / AI BOX 的 Home Agent Hub
- 明确 Debian 13 + Rust Backend + Angular WebUI 的基础技术方向
- 收敛 MVP 到摄像头主链路
- 落地 Phase 1 的 Rust schema 和 discovery 执行骨架

## 今日完成

### 1. 完成顶层架构与产品方向收敛

新增文档：

- `docs/home-agent-hub-harboros-ai-box-architecture.md`
- `docs/home-agent-hub-top-level-design.md`
- `docs/webui-information-architecture.md`

核心结论：

- 系统目标升级为 HarborOS / ARM-X86 AI BOX 双形态家庭智能中枢
- 运行环境固定为 `Debian 13`
- 后端主服务固定为 `Rust`
- WebUI 采用 `Angular`，并尽量对齐 `TrueNAS WebUI`
- 系统需同时包含普通用户前台和管理员后台

### 2. 完成 roadmap 与 phase backlog

新增/更新文档：

- `docs/home-agent-hub-roadmap.md`
- `docs/home-agent-hub-phase-backlog.md`
- `docs/home-agent-hub-top-level-design.md`

MVP 主链路正式收敛为：

`局域网自动扫描 -> 识别可用 RTSP 摄像头 -> 抓帧 -> AI 检测 -> 截图推送到 IM`

关键策略：

- 发现层使用 `ONVIF / SSDP / mDNS / RTSP Probe`
- 媒体层以 `RTSP` 为 MVP 主协议
- 明确要求发现过程不依赖手工填写摄像头 IP
