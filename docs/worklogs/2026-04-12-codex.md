## 今日完成

### 1. 补齐浏览器内实时观看链路

已在本地管理 API 中新增：

- `GET /live/cameras/<device_id>`
- `GET /api/cameras/<device_id>/live.mjpeg`

实现方式：

- 直接复用本地 RTSP 地址
- 由本机 `ffmpeg` 转成 `MJPEG` 长连接
- 浏览器端用 `<img>` 持续接收 `multipart/x-mixed-replace` 码流

结果：

- 不再依赖“反复抓拍 + 前端轮询”伪实时方案
- 手机浏览器可直接打开实时画面
- 已本地验证 HTML 页面和 MJPEG 流都能正常返回

### 2. 飞书 Bot 补上实时观看入口

已完成：

- 新增文字命令：`实时观看客厅摄像头`
- 在摄像头控制卡片中新增 `实时观看` 按钮
- Bot 会回一个可直接打开的浏览器链接，而不是只给出抓拍结果

当前局域网内观看入口示例：

- `http://192.168.3.30:4174/live/cameras/cam-rtsp-192-168-3-73`

### 3. 把“外网观看”补成安全分享链路

已新增：

- `src/runtime/remote_view.rs`
- 带签名、带过期时间的摄像头分享 token
- `GET /shared/cameras/<token>`
- `GET /shared/cameras/<token>/live.mjpeg`

设计原则：

- 外网用户只拿到临时分享链接
- 直连调试入口 `/live/cameras/*` 与 `/api/cameras/*` 默认只允许本机或局域网访问
- 外网分享链接过期后自动失效
- 分享密钥持久化到 `.harborbeacon/admin-console.json`

当前默认分享配置：

- `share_link_ttl_minutes = 120`

### 4. 明确“没有公网入口时不生成假链接”

已在 Bot 中补上判断：

- 若未配置 `HARBOR_EXTERNAL_ORIGIN`
- 或当前值仍是私网/`.local`/`localhost`

则：

- `外网观看客厅摄像头`
- 卡片中的 `外网分享`

都会返回明确报错，而不是给出一个实际无法在局域网外访问的假地址。

这样可以避免真实部署阶段产生错误预期。
